# release-check

## 목적
배포 직전 누락과 리스크를 줄이기 위한 최종 점검 절차를 제공한다.

## 사용 시점
- 배포 전
- 스테이징 점검
- 릴리즈 후보 검토

## 절차
1. 핵심 기능이 정상 동작하는지 확인한다.
2. 환경 설정과 필수 변수 누락 여부를 확인한다.
3. 모바일/웹 분기, 반응형, 다크모드 영향을 확인한다.
4. 콘솔 오류, 네트워크 오류, 예외 처리 누락을 점검한다.
5. 필요한 문서나 변경 요약이 준비되었는지 확인한다.
6. 필요하면 `qa-agent`와 `docs-agent`를 사용한다.

## 결과물
- 배포 전 체크 결과
- 발견된 위험 요소
- 즉시 수정 항목
- 배포 후 모니터링 포인트
