# bugfix-flow

## 목적
버그를 빠르게 고치는 것뿐 아니라 원인과 회귀 위험까지 함께 관리한다.

## 사용 시점
- 오류 수정
- 화면 이상 동작 수정
- API 예외 수정
- 데이터 처리 오류 수정
- 스타일 깨짐 또는 상태 처리 버그 수정

## 절차
1. 현재 증상과 기대 동작을 정리한다.
2. 재현 조건과 영향 범위를 확인한다.
3. UI 문제면 `frontend-agent`, 서버/데이터 문제면 `backend-agent`를 우선 고려한다.
4. 가장 작은 수정으로 원인을 해결한다.
5. 수정 범위와 부작용 가능성을 점검한다.
6. `qa-agent`로 회귀 여부를 검토한다.
7. `docs-agent`로 원인과 수정 내용을 간단히 정리한다.

## 결과물
- 원인 설명
- 수정 코드
- 영향 범위 메모
- 회귀 점검 결과

## 예외
- 재현이 불가능한 경우, 추정 수정으로 단정하지 말고 원인 후보와 추가 확인 포인트를 남긴다.
- 구조 변경이 필요한 경우, 간단한 버그 수정으로 위장하지 말고 범위를 재정의한다.
