# verify-change

## 목적
구현이 끝난 변경사항을 품질 관점에서 점검한다.

## 사용 시점
- 기능 구현 직후
- 버그 수정 직후
- PR 전 셀프 리뷰
- 사용자 전달 전 점검

## 절차
1. `.cursor/rules/60-delivery-gates.mdc` Gate 3(DoD) 관점에서 완료 조건을 충족하는지 확인한다.
2. 요구사항 대비 구현 누락이 없는지 확인한다.
3. 기본 상태, 로딩, 빈 상태, 오류 상태를 점검한다.
4. 모바일/웹 분기와 반응형을 점검한다.
5. 다크모드 영향을 점검한다.
6. 기존 기능에 영향이 없는지 회귀 위험을 확인한다.
7. 필요하면 `qa-agent`를 사용해 체크리스트 방식으로 정리한다.
8. 수정이 필요하면 다시 적절한 Skill 또는 Agent로 되돌린다.

## 결과물
- 검증 결과 요약
- 발견된 이슈 목록
- 수정 필요 항목
