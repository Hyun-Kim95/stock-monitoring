# document-change

## 목적
구현 결과를 사용자 또는 팀이 이해할 수 있는 형태로 정리한다.

## 사용 시점
- 작업 완료 후 공유
- README 업데이트
- 인수인계 문서 작성
- 변경 이력 정리
- 병렬 구현 중 API 계약·스키마·오류 규칙이 바뀌었을 때(즉시 동기화)

## 절차
1. 병렬 작업 중이면 최신 API·UI 계약과 문서가 일치하는지 먼저 확인한다.
2. 무엇이 바뀌었는지 정리한다.
3. 왜 바꿨는지 정리한다.
4. 어떤 파일이나 영역이 영향을 받았는지 정리한다.
5. 추가 확인이 필요한 부분을 표시한다.
6. 필요하면 `docs-agent`를 사용해 문서를 다듬는다.

## 결과물
- 변경 요약
- 영향 범위
- 사용 방법 또는 확인 포인트
- 후속 작업 메모
