# 모바일·반응형 대시보드 요구사항

## 브레이크포인트

| 구간 | 뷰포트 너비 | 레이아웃 |
|------|-------------|----------|
| 모바일 | `max-width: 979px` | 단일 열 스크롤. 헤더 압축, 상세 필터는 시트(`dialog`). |
| 태블릿 | `640px ~ 979px` | 본문 그리드: 관심종목 전체 너비 1행, 테마·뉴스 2열. |
| 데스크톱 | `min-width: 980px` | 기존과 동등한 3열(관심종목·테마·뉴스). `minmax(560px, …)` 제거해 좁은 창에서도 페이지 전체 가로 스크롤을 유발하지 않음. |

설정 UI 레이아웃(`(admin)` 그룹) 기존 `980px` 기준과 정렬한다.

## 정보 구조 (IA)

1. **상단**: 제목, 연결 상태(짧은 표시), 테마 토글, 설정 링크.
2. **모바일**: 검색(종목명·코드·별칭)은 헤더 아래 전폭 행. 시장·세션·NXT·등락률 알림은 **필터** 버튼으로 여는 모달에서 조작.
3. **본문 순서**: 관심종목(표·선택 시 차트 포함) → 테마 → 관련 뉴스.
4. 데스크톱에서는 필터·알림을 헤더 도구줄에 유지.

## 테이블

- 웹 목록 규칙(필터 상단, 페이지네이션 등)은 유지하되, 관심종목 표는 **좁은 폭에서 가로 스크롤** 허용(`overflow-x: auto`, 스티키 헤더 유지).
- 행 확장·컬럼 숨김은 후속 과제로 두고, 1차 범위에서는 가로 스크롤로 대응.

## 상태

- 기본 / 로딩 / 빈 / 오류는 기존 메시지 유지. 좁은 폭에서도 레이아웃이 깨지지 않아야 한다.
- 라이트·다크 모두 동일 구조.

## 접근성

- 필터 시트는 제목·닫기 버튼 제공, 포커스 가능한 컨트롤 유지.
- 터치 영역은 가능한 한 **최소 약 44px**에 가깝게(버튼·행 간격).
