# 플랫폼 운영자 사이트 QA 체크리스트

기준: [`../platform-operator-site-prd.md`](../platform-operator-site-prd.md) §4·§5·§7·§8·§10. 계약 상세: [api-contract.md](./api-contract.md).

## 인증·권한 (PRD §10-1, §2, §7)

- [ ] 비로그인으로 플랫폼 웹·`/platform/*` 호출 시 **401** / 로그인 유도.
- [ ] 로그인했으나 **전역 운영자 플래그 없음** → 웹 리다이렉트, API **403**.
- [ ] 테넌트 **`OWNER`/`ADMIN`만** 있고 플래그 없을 때 플랫폼에 **접근 불가**(PRD §0).
- [ ] **`ADMIN_API_TOKEN` Bearer만**으로는 플랫폼 쓰기·읽기가 **통과하지 않음**.
- [ ] (도입 시) **`PLATFORM_API_TOKEN`** 만 허용된 경로가 계약과 일치하고, 테넌트 쓰기에 **`tenantId` 누락 시 400**인지.

## 테넌트 컨텍스트·실수 방지 (PRD §0·§4.1·§6)

- [ ] 테넌트 스코프 화면에 **선택된 `tenantId`** 가 항상 보인다.
- [ ] API 요청이 **경로/쿼리의 `tenantId`** 를 사용하는지(세션 테넌트만으로 민감 데이터 조회하지 않음).
- [ ] UI에서 다른 테넌트로 바꾼 뒤 이전 테넌트 데이터가 **남지 않음**(스테일 상태 방지).
- [ ] **1차 네비게이션**(지원 · 데이터·수집 · 선택 구역)이 PRD §4.1 IA와 맞는지.

## 사용자 지원 S-01 ~ S-03 (PRD §10-2)

- [ ] 사용자 검색: 이메일·표시명·id — **빈 검색** 시 동작이 계약과 일치(**400** 또는 비활성).
- [ ] 검색 결과 **과다** 시 상한·안내가 [api-contract.md](./api-contract.md)와 일치.
- [ ] 사용자 상세에 **멤버십 전체** 표시; 멤버십 0명 시 빈 안내(§8.2).
- [ ] 다중 멤버십 사용자에서 “세션 테넌트 = 유일 테넌트” **오해가 나지 않음**(PRD §3).
- [ ] **S-02** 프로필: 가입일·OAuth 요약이 표시되고 민감 필드는 §5 정책과 일치.

## 문의 S-04 ~ S-05 (PRD §10-3)

- [ ] 테넌트·기간·(선택) 키워드 필터; `from` > `to` → **400**.
- [ ] 문의 상세·목록에서 작성자 삭제 등 **깨진 참조** 시 플레이스홀더(§8.3).
- [ ] 답변 등록 후 목록·상세에 반영; 본문 길이 초과 **400**.
- [ ] 답변 버튼 **중복 제출 방지**; 서버 **60초 내 동일 본문 409** 동작( [api-contract.md](./api-contract.md) ).
- [ ] 문의 **상세**에 기존 **답변 스레드**(시간순)가 보이는지([api-contract.md](./api-contract.md) 응답 메모).

## 테넌트·설정 D-01·D-02 (PRD §10-4)

- [ ] 테넌트 검색·목록 **페이지당 15건**·필터·하단 페이지네이션·검색 옆 **초기화**(§4.5, 규칙 30).
- [ ] 존재하지 않는 `tenantId` → **404**(또는 합의한 정책).
- [ ] `SystemSetting` 조회·갱신; 고위험 키는 **이중 확인**·마스킹(§5).
- [ ] 동시 수정 시 **`expectedUpdatedAt` 불일치 → 409**; UI 새로고침·재시도; 설정 UI와 동시 편집 시나리오 1회 이상.

## 카탈로그 요약 D-03 (도입 시)

- [ ] `GET .../catalog-summary`(또는 동등)로 종목·테마·규칙 **집계만** 노출되고, CRUD는 `/settings`로 유도되는지(PRD §4.3 D-03).

## 수집 건강도 D-04 (도입 시)

- [ ] 종목 0건 테넌트는 **빈 상태**로 오류 아님.
- [ ] 히스토리 없는 종목 **200 빈**; 상류 실패 시 **503** 등 계약대로 메시지.

## 비기능·오류 매핑 (PRD §7·§8, §10-7)

- [ ] **401·403·400·404·409·429·500·503** 및 `error.code`가 사용자(운영자)에게 구분되어 보인다.
- [ ] **429** 시 `error.code`가 **`RATE_LIMIT`**(제품 공통)인지.
- [ ] **세션 만료** 중 폼 작성 → 401 후 로그인 유도 및 손실 안내(§7.1).
- [ ] 네트워크 타임아웃 시 재시도·안내(§7.1).
- [ ] **429** 발생 조건을 스테이징에서 재현 가능한지.
- [ ] **correlation id** 헤더가 있으면 로그와 대조 가능.
- [ ] (동일 CSRF 정책 시) 답변·설정 저장에서 **403 `CSRF_REJECTED`** / **503 `CSRF_CONFIG`** 재현 및 [operations.md](./operations.md) 대응과 일치.

## 개인정보·고위험 §5

- [ ] 이메일·문의 본문 **마스킹·최소 노출**이 디자인·구현과 일치.
- [ ] `SystemSetting` 등 **고위험 변경**에 확인 모달 + 감사 로그(§5·§10-6).

## 감사 A-01 (PRD §10-6)

- [ ] 민감 조회·쓰기 후 감사에 **운영자·대상·tenant·출처 채널**이 남는지(저장 형태는 구현 SSOT).
- [ ] 선택한 감사 실패 정책(a/b/c)이 [operations.md](./operations.md)와 실제 동작이 일치.

## UI·접근성·테마 (PRD §4.1, 규칙 40)

- [ ] 라이트/다크 모드 전환 및 **재방문 후 유지**.
- [ ] 로딩·빈·오류·권한 상태가 화면마다 일관.

## 보안 A-02 (권장)

- [ ] 운영 환경에서 **2FA 또는 IP 허용 목록** 적용 여부가 [operations.md](./operations.md)·인프라 설정과 일치.

## 로드맵 (비MVP — PRD §4.2 S-06, §4.3 D-05)

- [ ] S-06 세션 무효화·D-05 백필·감사 UI가 **범위에서 제외**된 경우 문서·릴리즈 노트에 명시.

## PRD §9 확정 정합

- [ ] 구현·[api-contract.md](./api-contract.md)·[operations.md](./operations.md)가 PRD §9 **확정(추천안 채택)** 과 불일치하면 문서 또는 코드를 먼저 맞춘다.
