# changelog · 2026-05

## 2026-05-14 — 플랫폼 운영자 사이트 IA 재구성

### 변경 내용

- IA를 객체 중심(테넌트/사용자)에서 **작업 중심 평면 메뉴**로 재구성: 대시보드 · 회원 · 문의 · 공지 · 테넌트 · 감사 로그(6개).
- 공지 모델(`Announcement`) 신설 + 운영자 콘솔 CRUD(작성·수정·발행·취소·삭제). 사용자 노출 UI는 다음 사이클.
- 신규 백엔드 엔드포인트 5종: `GET /platform/overview`, `GET /platform/users`(기본 목록), `GET /platform/inquiries`(크로스 테넌트), `Announcements` CRUD(`/platform/announcements`), `GET /platform/audit-logs`.
- `apps/api/src/routes/platform.ts` 단일 파일을 `apps/api/src/routes/platform/` 디렉터리로 분할(tenants/users/inquiries/overview/announcements/audit/lib).
- `recordAuditLog` 헬퍼 도입. 설정 갱신·문의 답변·공지 쓰기 액션을 비즈니스 쓰기와 **동일 Prisma 트랜잭션**에서 감사 로그 INSERT(PRD §4.4 A-04).
- 프론트엔드 신규 페이지: `/platform`(대시보드 KPI), `/platform/users`(기본 목록 + 검색 통합), `/platform/inquiries`(크로스 테넌트 통합 목록), `/platform/announcements`·`/new`·`/[id]`(공지 관리), `/platform/audit-logs`(필터·페이지네이션 표). 모두 페이지당 15건·다크모드 토큰·반응형.

## 2026-05-14 — 권한 가드 강제 로그아웃 통일 + 사용자 문의 답변 라우트 일원화

### 변경 내용

- `useEnforceAccess` 훅 신설(`apps/web/src/hooks/useEnforceAccess.ts`). `(admin)`·`(platform)` 레이아웃이 동일 훅을 사용해 권한 평가가 끝날 때까지 본문(`children`)을 렌더하지 않는다. 권한 미충족 사용자는 즉시 `POST /auth/logout` 후 `/login?reason=forbidden`으로 리다이렉트.
- `apps/web/src/app/login/page.tsx`에 `reason=forbidden` 안내 배너 추가("접근 권한이 없어 로그아웃되었습니다.").
- `apps/web/src/app/(admin)/settings/inquiries/page.tsx`에서 답변 작성 폼·상태·`submitReply` 핸들러 제거. 답변 목록(읽기 전용)과 빈 상태 안내는 유지. 페이지 헤더 문구도 "운영자의 답변을 확인" 뉘앙스로 정정.
- `apps/api/src/routes/inquiries.ts`에서 `POST /inquiries/:id/replies` 핸들러와 `ReplyCreateSchema` 제거. 운영자 답변은 `POST /platform/inquiries/:inquiryId/replies`(플랫폼 운영자 콘솔)로 단일화.

### 영향 범위

- 웹: `(admin)`·`(platform)` 레이아웃 가드 패턴 통일, 로그인 페이지 안내 분기, 사용자 문의 화면 답변 UI 제거.
- API: 사용자 문의 답변 라우트 1종 제거(테넌트 `OWNER/ADMIN` `adminPre` 보호). 호출처는 동시 제거되어 다른 클라이언트는 영향 없음.
- 문서: 본 changelog 1건 추가.

### 확인 필요사항

- 외부 자동화가 `POST /inquiries/:id/replies`를 호출하고 있었다면 `/platform/inquiries/:inquiryId/replies`(플랫폼 운영자 세션 필요)로 교체 필요. 사내 사용 사례는 현재 없음.
- 로그인 페이지의 `reason` 쿼리는 본 사이클에서 `forbidden`만 처리. 향후 `expired` 등 확장 시 같은 분기에 추가.
