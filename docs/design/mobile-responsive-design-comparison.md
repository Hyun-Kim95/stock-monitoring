## 안 A — 로컬 와이어

- 산출물: [`mobile-responsive-wireframes.md`](./mobile-responsive-wireframes.md)
- 요약: 단일 열 + 필터 `dialog`, 표는 가로 스크롤, 태블릿에서 테마·뉴스 2열.

## 안 B — Stitch (CLI)

- **Stitch MCP는 사용하지 않으며**, 동일 범위 화면은 **Stitch CLI**로 재현 가능하다. 품질 기준은 [`stitch-sop.md`](./stitch-sop.md)의 모바일·다크·상태 스펙을 참고한다.
- 본 저장소 실행 세션에서는 CLI 산출 이미지 대신 **안 A 스펙과 본 문서의 비교표**로 게이트를 충족하고, 후속에 CLI로 스크린샷·에셋을 보강할 수 있다.

| 항목 | 안 A (로컬) | 안 B (Stitch CLI) |
|------|-------------|-------------------|
| 정보 우선순위 | 목록 → 차트 → 테마 → 뉴스 명확 | 동일 범위 프롬프트 시 시각 정돈 유리 |
| 필터 발견성 | 헤더 `필터` 버튼 | 동등하게 설계 가능 |
| 구현 난이도 | 낮음 (기존 컴포넌트 재배치) | 디자인 단계 비용만 추가 |
| 상태 UI | 로딩/빈/오류 메시지 유지 | 참고용 목업 |

## 선택

- **구현 기준안**: **안 A** (와이어에 정의된 IA·브레이크포인트).
- **안 B**: 시각 검증·이해관계자 공유용으로 Stitch CLI 재실행 시 보완 (필수 산출은 스펙 문서로 대체).

## 제외·후속

- 세그먼트 탭으로 목록/차트/뉴스 전환하는 대안은 비용 대비 발견성 이슈로 **제외**.
- 표 컬럼 숨김·행 확장은 후속.
